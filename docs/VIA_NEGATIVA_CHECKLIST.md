# VIA_NEGATIVA_CHECKLIST

This checklist is for `aoa-agents` as the role contracts and handoff
boundaries.

## Keep intact

- profiles with materially distinct role contracts
- explicit handoff rules and memory posture
- agent definitions that narrow ambiguity

## Merge, move, suppress, quarantine, deprecate, or remove when found

- profiles that differ only by style or narrative flavor
- agent-local routing or policy that duplicates other layer ownership
- handoff variants without distinct thresholds or authority changes

## Questions before adding anything new

1. Would one role contract plus style notes be enough?
2. Does this agent add a new boundary, or only a new voice?
3. Is this handoff shape already expressed elsewhere?

## Safe exceptions

- a real new role with new contract and evaluation posture
- temporary alias surfaces with migration notes

## Exit condition

- Surviving profiles should make the system easier to route and audit.
