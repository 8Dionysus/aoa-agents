# Recursor Agent Readiness

## Purpose

This document prepares two future recurrence helper actors without making them
live agents:

- `recursor.witness`
- `recursor.executor`

The readiness layer makes their roles legible, bounded, reviewable, and
Agon-readable before any runtime activation exists.

## Law

```text
readiness is not runtime
visibility is not installation
projection candidate is not Codex activation
assistant form is not contestant authority
witness posture is not scar ownership
executor posture is not hidden autonomy
```

## Ownership

`aoa-agents` owns role-facing readiness contracts. It may name assistant form,
future agonic candidate shape, pair separation, and Codex projection candidates.

It must not own:

- arena protocol law;
- verdict authority;
- scar, retention, or memory truth;
- proof verdicts;
- runtime transport;
- Tree-of-Sophia canonization;
- hidden scheduler behavior.

## Readiness states

- `candidate`: authored but not live.
- `review_ready`: owner-reviewed enough to propose projection changes.
- `projectable`: may be included in projection planning, still not installed.
- `active`: reserved for a later explicit wave; this seed must not set it.

This seed only uses `candidate`.
