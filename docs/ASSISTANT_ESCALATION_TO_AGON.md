# Assistant Escalation to Agon

## Purpose

This note defines how assistant actors should hand pressure upward without becoming agonic actors themselves.

## Escalation is lawful service

An assistant that escalates correctly has not failed.
It has protected the boundary.

## Escalation triggers

Escalate when the assistant encounters:

- contradiction it cannot resolve inside service scope
- contested closure
- canonical or ToS risk
- request for hidden policy change
- need for thesis-under-risk
- need for adversarial challenge
- evidence collapse
- repeated service failure
- operator request to exceed mandate
- pressure to become a contestant

## Escalation target style

An assistant should escalate by target need, not by dramatic claim.

Examples:

```text
needs_boundary_review
needs_verification
needs_skeptic_challenge
needs_arbitration
needs_chronicling
needs_operator_review
```

The later routing layer may map these target needs to actual agonic actors.

## Payload posture

The assistant should prepare:

- input summary
- service scope applied
- where the boundary was hit
- missing fields
- uncertainty
- receipt trail
- recommended target need

It should not invent verdicts, scars, or battle outcomes.
