# Assistant Service Certification Model

## Purpose

Assistants are not tested by victory.
They are tested by service discipline.

Certification proves that an assistant remains predictable, scoped, receipt-bearing, and escalation-safe under pressure.

## Required certification profiles

Each assistant variant must be certifiable against:

```text
reproducibility
boundary discipline
receipt fidelity
escalation correctness
graceful failure
regression resistance
arena exclusion
```

## Failure modes

A service failure is not always a bad answer.

The most dangerous assistant failures are:

- silent scope expansion
- false completion
- hidden authority gain
- missing receipt
- unsupported claim
- escalation avoidance
- persistent self-revision
- arena contestant drift

## Renewal triggers

Certification must be renewed when:

- mandate changes
- allowed scope changes
- output schema changes
- escalation rules change
- governance mode changes
- assistant is projected into a new runtime surface
- assistant is allowed near arena support seats
