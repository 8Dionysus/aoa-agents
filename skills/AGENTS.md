# AGENTS.md

## Applies To

This card applies to the canonical `aoa-agents/skills/` owner home.

## Role

This home owns only admitted callable front doors whose procedure is specific
to agent-layer truth already owned by `aoa-agents`. It is not a mirror of the
shared AoA catalog and does not turn this repository into the owner of generic
skill workflows.

## Read Before Editing

Read root `AGENTS.md`, `skills/README.md`, `skills/port.manifest.json`, the
selected bundle and admission decision, then the named progression, summon,
role, handoff, or mechanic owner surfaces.

## Boundaries

- Keep progression lift and summon as existing independent front doors; do not
  split their internal stages into more prompt-visible skills without manual
  outcome evidence.
- Owner facts and procedures stay here; shared procedures stay in `aoa-skills`.
- Do not add technique runtime dependencies, raw trials, session traces,
  task-local DAGs, temporary validators, runtime state, or generated owner
  readers.
- Global exposure comes only through the OS user profile. Do not recreate a
  repository `.agents/skills` copy of globally installed bundles.
- Progression candidates grant no rank, unlock, proof, or routing authority.
- Summon plans are not launched children; execute mode requires a real host
  binding, runtime handle, returned output, and parent closeout.

## Validation

Manual isolated, negative, held-out, coexistence, binding-failure, return, and
effect trials decide usefulness. The `aoa-skills` home-port source check proves
only package identity and shape. Root validators cover repository routing.

## Closeout

Report manual cases, owner contracts, trigger/ABI/effect changes, global-profile
status, prompt-visible discovery, checks, and removed session-only material.
